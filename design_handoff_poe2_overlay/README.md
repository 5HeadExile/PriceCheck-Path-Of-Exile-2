# Handoff: Редизайн UI оверлея PriceCheck PoE2

## Overview
Освежение двух десктоп-поверхностей оверлея для Path of Exile 2:
**игровое меню** (быстрые действия) и **окно настроек**. Тёмная тема,
премиальная приглушённая палитра в духе PoE: воронёная сталь + сдержанное
золото-акцент. Цель — компактный, не громоздкий, цельный вид.

Визуальное направление: **«Воронёная сталь и сдержанное золото»**. Тёмные
металлические поверхности с тонкими гравированными рамками; золото —
редкий, дорогой акцент, появляется ТОЛЬКО на первичном действии, фокусе и
состоянии «активно». Премиальность достигается глубиной (градиенты,
внутренние тени, рамки), а не яркостью.

## About the Design Files
Файлы в этом бандле — **дизайн-референс, сделанный в HTML** (прототип,
показывающий желаемый вид), а НЕ продакшн-код для копирования. Задача —
**воссоздать этот вид в существующем коде проекта**.

⚠️ Целевая среда: **WinForms (.NET) + отрисовка в GDI+**. Без WPF, без Web/CSS,
без сторонних UI-библиотек. Все элементы рисуются кодом в `OnPaint`
(`Graphics`, `GraphicsPath`, `LinearGradientBrush`, `SolidBrush`, `Pen`).
HTML здесь — только эталон пикселей, цветов и состояний. Раздел
**"GDI+ Implementation Notes"** ниже показывает, как перевести это в C#.

## Fidelity
**High-fidelity (hifi).** Цвета, размеры, отступы и состояния финальные.
Воссоздавать пиксель-в-пиксель значениями из раздела Design Tokens.

## Interactions & Behavior
- **Hover**: фон `surface→surface.hover`, текст `muted→text`, рамка
  `quiet→strong`. Переход мгновенный (GDI+ перерисовка по `MouseEnter/Leave`).
  Можно лёгкий fade 80–120ms через таймер, но не обязательно.
- **Focus** (Tab-навигация): золотая рамка + свечение. Обязательно для
  доступности с клавиатуры.
- **Active**: вдавливание (тёмный фон + внутр. тень) на `MouseDown`.
- **Слайдер**: перетаскивание ручки обновляет заливку и чип-значение в
  реальном времени.
- **Keycap**: клик переводит в режим «нажмите клавишу», следующее нажатие
  записывается как хоткей.
- **«Оверлей: пауза»**: тоггл; тег `[ВКЛ]/[ВЫКЛ]` и иконка ⏸/▶ отражают
  состояние.
- **Выход**: опасное действие, подтверждение по желанию.
- **Настройки** футер: «Сохранить» применяет и закрывает; «Отмена»
  отбрасывает изменения.

## State Management
- `overlayState`: `active | paused` → иконка и тег в меню.
- `selectionState`: есть ли выделенная область (влияет на доступность
  «Сбросить»).
- Настройки (персистентно, напр. в `%AppData%`): `league`, `priceRefreshMin`,
  `hotkeyMenu`, `hotkeyCalibrate`, `menuDim`, `overlayOpacity`, `pricesUrl`,
  `ocrThreshold`, `ocrDebug`.
- Слайдеры/степперы двусторонне связаны со значениями.

## GDI+ Implementation Notes (WinForms → C#)
- **Скруглённые прямоугольники**: helper `GraphicsPath RoundedRect(Rectangle r, int radius)`
  (4 дуги `AddArc` + `CloseFigure`). Использовать для окна, кнопок, полей, чипов.
- **Включить сглаживание**: `g.SmoothingMode = SmoothingMode.AntiAlias;`
  `g.TextRenderingHint = TextRenderingHint.ClearTypeGridFit;`
- **Вертикальный градиент окна/кнопок**: `LinearGradientBrush` с
  `LinearGradientMode.Vertical`.
- **Заливка слайдера**: `LinearGradientBrush` горизонтальный `#9C7A30→#D6B668`,
  клиппинг по ширине заполнения.
- **Рамки**: `using var pen = new Pen(ColorTranslator.FromHtml("#2E2E36"), 1f);`
  рисовать по тому же `GraphicsPath`.
- **Свечение фокуса**: дополнительный `Pen` 2px полупрозрачным `Color.FromArgb(56,201,162,75)`
  вокруг (или `DrawPath` по чуть большему пути).
- **Тень окна**: при borderless — несколько полупрозрачных смещённых прямоугольников,
  либо `DropShadow` через `CS_DROPSHADOW` в `CreateParams`.
- **Кастомные контролы**: наследовать `Control`, переопределять `OnPaint`,
  `OnMouseEnter/Leave/Down/Up`, `OnGotFocus/LostFocus`; хранить флаг состояния
  и звать `Invalidate()`.
- **Иконки**: `Font iconFont = new Font("Segoe MDL2 Assets", 12f);` и рисовать
  glyph-символ (см. таблицу глифов), либо рисовать вручную (как кольцо питания
  для «Выхода»).
- **Глиф «Выход» вручную**: дуга кольца `DrawArc` с разрывом сверху +
  вертикальная черта `DrawLine` — цвет `danger`.
- Скрывать дефолтную рамку формы (`FormBorderStyle.None`) и рисовать титулбар/
  логотип/кнопку закрытия самим; обрабатывать перетаскивание окна по титулбару
  (`WM_NCLBUTTONDOWN` / ручной drag).
