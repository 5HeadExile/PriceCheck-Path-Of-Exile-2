# Бриф для редизайна UI — PriceCheck PoE2

Документ для дизайн-сессии (Claude Design). Цель: освежить **игровое меню** и
**окно настроек**. Стиль — интересный и красивый, но **не громоздкий**, компактный.

---

## 1. Что это за приложение

Десктоп-оверлей для игры **Path of Exile 2** (Windows). Запускается в трей, поверх
игры (windowed-fullscreen). Основная функция — по открытию «пилона наград» сам
распознаёт награды (OCR области экрана) и рисует **цену напротив каждой строки**
награды (источник цен — poe.ninja). Игрок выбирает **одну** награду из списка.

Сейчас в продукте **одна фича — оценка пилонов**. Вторую (price-check предметов)
временно вырезали, к ней вернёмся позже — на дизайн это не влияет.

## 2. Технические ограничения (важно для реализуемости)

- **Стек:** C# / .NET 8, **WinForms**, UI собирается **кодом** (без XAML, без
  Web/CSS, без дизайнера). Рисование — стандартные контролы WinForms + кастомный
  `OnPaint` (GDI+: `Graphics`, `SolidBrush`, `Pen`, `GraphicsPath`, скругления).
- **Без тяжёлых UI-библиотек.** В проекте есть MahApps.Metro, но он не используется;
  тащить веб-стек/WPF не хотим. Решение должно ложиться на GDI+ или флэт-контролы.
- **Тёмная тема обязательна** — оверлей висит поверх тёмной игры.
- **Локализация — русский** (тексты кнопок/полей на русском).
- **DPI:** Per-Monitor V2 (учитывать масштаб; не завязываться на пиксель-перфект).
- **Оверлей цен — click-through, борд-лесс, TopMost.** Нюанс: ключ прозрачности
  чёрный, текст рисуем серым AA (не ClearType — на прозрачном окне даёт цветную
  кайму). Это про оверлей цен; меню/настройки — обычные окна, тут свободы больше.
- Анимации возможны только простые (таймер + перерисовка). Без сложного motion.

## 3. Поверхности для редизайна

### 3.1 Игровое меню (открывается по хоткею F2 или даблкликом по трею)

Сейчас: затемнение всего экрана (полупрозрачный чёрный, opacity ~0.55) + по центру
**борд-лесс окно 320×380**, `BackColor #18181C`. Структура:

- Заголовок: «PriceCheck PoE2», Segoe UI 13 Bold, по центру, высота 56.
- Вертикальный список из **7 плоских кнопок** (272×36, отступ сверху 6,
  `BackColor #282830`, рамка `#464650`, текст Gainsboro, FlatStyle.Flat):
  1. «Выделить область» (калибровка области пилона)
  2. «Сбросить выделенное»
  3. «Оверлей: пауза / возобновить»
  4. «Пересканировать сейчас»
  5. «Настройки»
  6. «Закрыть меню»
  7. «Выход»

Закрытие: повтор хоткея, Esc или клик по затемнению.
Проблемы текущего вида: выглядит как дефолтный WinForms-список кнопок, нет иерархии
(все кнопки равнозначны), нет иконок/группировки, «Выход» визуально как всё
остальное (нет акцента/опасного действия), заголовок занимает много места.

### 3.2 Окно настроек (пункт меню «Настройки»)

Сейчас: `FixedDialog`, `CenterScreen`, **440×480**, дефолтная светлая тема WinForms
(не стилизовано!). `TableLayoutPanel` 2 колонки (лейбл 150px + контрол). Поля:

| Лейбл | Контрол | Дефолт |
|---|---|---|
| Лига | TextBox | «Runes of Aldur» |
| URL цен (poe.ninja) | TextBox | poe.ninja экономика |
| Обновление цен, мин | NumericUpDown (1..240) | 30 |
| Хоткей меню | TextBox | VcF2 |
| Хоткей калибровки | TextBox | VcF4 |
| Затемнение меню, % | TrackBar (0..100) | 55 |
| Прозрачность оверлея, % | TrackBar (0..100) | 90 |
| Порог OCR (0..255) | NumericUpDown | 110 |
| Отладка OCR | CheckBox | выкл |

Внизу справа кнопки «Сохранить» / «Отмена» (по 100px).
Проблемы: окно **не в тёмной теме** (контраст с меню/игрой), плоский список полей
без группировки (Общие / Хоткеи / Внешний вид / Отладка), технические значения
(порог OCR, URL) показаны наравне с частыми, нет подсказок/единиц рядом со
слайдерами (не видно текущее число у TrackBar).

### 3.3 Трей-меню (правый клик по иконке) — второстепенно

Стандартный `ContextMenuStrip`: «Открыть меню» / «Настройки» / — / «Выход».
Можно слегка причесать, но это нативное меню Windows — глубокий редизайн не нужен.

## 4. Существующий визуальный язык (для консистентности)

Оверлей цен (его НЕ трогаем, но дизайн меню/настроек должен с ним рифмоваться):
- Тёмные **скруглённые плашки** (радиус 8), фон `#18181C`, рамка `#3A3A42`.
- Текст — Segoe UI 10.5 Bold.
- **Цвета по «тиру ценности»**: от серого (дёшево) к розово-красному (дорого).
- Лучшая награда: мягкая **золотая рамка** `#D6B668` + префикс «★».
- Палитра приглушённая, премиальная, без кислотных цветов.

## 5. Что хотим получить от дизайна

1. **Визуальное направление** (одно, конкретное — не «чисто-минимал»): напр. dark
   luxury в духе PoE, со сдержанным золотом-акцентом; или тёмное стекло; решает
   дизайнер, но осознанно.
2. **Токены**: палитра (hex: фон/поверхности/текст/акцент/опасное действие/границы),
   типошкала (2 размера-3), сетка отступов, радиусы, толщины рамок.
3. **Компоненты с состояниями**: кнопка (обычная/hover/focus/active/опасная),
   поле ввода, слайдер с подписью значения, чекбокс, заголовок секции.
4. **Раскладка меню**: компактнее, с иерархией (первичные действия выделены,
   «Выход» — как опасное/второстепенное), по возможности иконки-глифы (Segoe MDL2/
   юникод), уменьшить «пустой» заголовок.
5. **Раскладка настроек**: тёмная тема, группировка по секциям (Общие · Хоткеи ·
   Внешний вид · Отладка), у слайдеров видно текущее число, «продвинутые» поля
   (URL, порог OCR) визуально приглушены/свёрнуты.
6. **Мокапы** (ASCII/таблицей/описанием по пикселям) обеих поверхностей + краткое
   обоснование. **Реализуемо в WinForms GDI+** — без веб-компонентов.

Размер окон желателен компактный: меню ~300–340 шириной, настройки ~420–460.

## 6. Анти-цели

- Не делать «дашборд с сайдбаром и карточками».
- Не раздувать окна, не плодить вкладки ради вкладок.
- Не предлагать решения, требующие WPF/Electron/Web.
- Не ломать читаемость поверх яркой игровой сцены (контраст, тени-подложки).
