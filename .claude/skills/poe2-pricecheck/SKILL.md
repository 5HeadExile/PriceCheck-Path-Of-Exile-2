## Цель и статус

**Цель.** Десктоп-приложение (оверлей) для оценки наград в новой лиг-механике
PoE2 — пилоны с наградами. Считает суммарную ценность наград каждого пилона,
помогает выбрать выгодный, показывая цены поверх игры.

**За основу взят** открытый проект
[PoeAncientsPriceHelper](https://github.com/pedro-quiterio/PoeAncientsPriceHelper)
(C#/.NET 8 оверлей, Tesseract OCR + poe.ninja). Дорабатываем и улучшаем.

**Текущий статус:** план реализован по коду целиком (M0–M7). Осталось то, что
требует ПК пользователя: сборка/тесты на Windows, сверка endpoint poe.ninja в
браузере и калибровка порога OCR на реальных скриншотах. Сборка на Linux
невозможна (см. «Лог ошибок»).

**Текущая лига PoE2:** «Return of the Ancients» (0.5.0), механика — «Runes of
Aldur» (это и есть пилоны). На poe.ninja slug лиги — `runesofaldur`, имя для
API — «Runes of Aldur» (дефолт в конфиге).

**План целиком:** `/root/.claude/plans/floofy-wiggling-elephant.md` (утверждён).

## Решения (и почему)

- **Стек C#/.NET 8 (WPF + WinForms)** — максимальное переиспользование архитектуры
  референса, быстрый старт. Минус: только Windows (пользователь согласен).
- **Источник цен — poe.ninja (раздел PoE2).** Заложен интерфейс `IPriceSource`,
  чтобы при необходимости заменить на офиц. Trade API PoE2. Точный endpoint PoE2
  ещё не подтверждён — уточнить инспекцией сетевых запросов poe.ninja/poe2/economy.
- **Распознавание — OCR-оверлей** с разовой калибровкой области экрана.
- **Новая фича: запуск в трей + игровое меню по хоткею.** Приложение стартует
  свёрнутым в трей; меню открывается только глобальным хоткеем прямо в игре —
  экран затемняется полупрозрачным слоем, по центру всплывает окно меню; повтор
  хоткея/Esc закрывает. Требует windowed-fullscreen в игре.
- **Зависимости как в референсе:** Tesseract 5.2.0 (+English 4.0.0), SharpHook
  7.1.2, MahApps.Metro 2.4.10, Newtonsoft.Json 13.0.3.
- **Окно настроек — WinForms, не WPF/MahApps** (отклонение от плана). Причина:
  единый UI-стек с треем/оверлеем и надёжная сборка непроверенного кода; XAML/
  MahApps-полировку можно вернуть позже. Зависимость MahApps пока оставлена.
- **Endpoint цен вынесен в конфиг** (`PriceApiBaseUrl`, `PriceOverviews`),
  дефолт — `https://poe.ninja/poe2/api/economy/currencyexchange/overview`.
  Парсинг ответа устойчив к разным формам (lines/items, exalted/chaos/value).

## Инструменты по моментам

| Задача | Инструмент / подход |
|---|---|
| Глобальные хоткеи (ловятся в игре) | `SharpHook` (low-level хук) |
| Трей-иконка, запуск без окна | WinForms `NotifyIcon` + `ApplicationContext` |
| Затемнение экрана + меню | Полупрозрачные WinForms-формы (TopMost, borderless) |
| Захват области экрана | GDI `BitBlt` / `Graphics.CopyFromScreen` |
| OCR | `Tesseract`, предобработка: grayscale + threshold + upscale (тёмный фон) |
| Нечёткое сопоставление имён наград | Левенштейн поверх `Data/reward-aliases.json` |
| HTTP к poe.ninja | `HttpClient` + `Newtonsoft.Json` |
| Окно настроек | WPF + `MahApps.Metro` |
| DPI | `app.manifest`, Per-Monitor-V2 |

## Лог ошибок (что → как решил → как не повторять)

- **Окружение сборки.** Контейнер Claude Code на Linux, `dotnet` отсутствует, а
  проект таргетит `net8.0-windows` (WPF/WinForms нельзя собрать на Linux).
  *Решение:* пишем исходники здесь, сборку/тесты (`dotnet build`/`dotnet test`)
  выполняет пользователь на Windows. *Впредь:* не обещать «собрано/проверено» из
  этого окружения; явно помечать Windows-only шаги как непроверенные локально.
- **poe.ninja PoE2 endpoint не подтверждается из песочницы.** Прямые запросы к
  `poe.ninja/poe2/api/...` дают 404 (сайт на Next.js, нужны браузерные заголовки/
  buildId-путь). *Решение:* endpoint вынесен в конфиг, парсинг устойчив к формам;
  финальную сверку делать в браузере (DevTools → Network на poe.ninja/poe2/economy).
  *Впредь:* не «подтверждать» внешний API по 404 из песочницы — помечать как
  требующий проверки и делать заменяемым.
- **Тесты на internal-методы.** `RewardParser.Levenshtein` и
  `PylonScanner.SplitCount` — `internal`; тесты в другой сборке. *Решение:*
  `InternalsVisibleTo("PriceCheckPoe2.Tests")` в `Properties/AssemblyInfo.cs`.
  *Впредь:* при тестировании internal-логики сразу добавлять этот атрибут.
